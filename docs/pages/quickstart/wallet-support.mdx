# For Wallet Developers

Follow this guide to support a polished experience for your users on Tempo. Integrate with [Tempo Transactions](/guide/tempo-transaction) to provide enhanced features like gas sponsorship and more with a few lines of code.

:::warning
Because there is [no native token on Tempo](/quickstart/evm-compatibility#handling-eth-native-token-balance-checks), wallets accustomed to native token logic should adjust accordingly for their users on Tempo.
:::

## Steps

::::steps

### Handle native token balance checks

If you use `eth_getBalance` to validate a user's balance, you should instead check the user's account fee token balance on Tempo. Additionally, you should not display any "native balance" in your UI for Tempo users.

:::info
In testnet, `eth_getBalance` [returns a large placeholder value](/quickstart/evm-compatibility#handling-eth-native-token-balance-checks) for the native token balance to unblock existing assumptions wallets have about the native token balance.
:::

:::code-group

```ts twoslash [example.ts]
import { client } from './viem.config'

const userFeeToken = await client.fee.getUserToken({ 
  account: '0x...' 
})

const balance = await client.token.getBalance({ 
  account: '0x...',
  token: userFeeToken.address 
})
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/viem.config.ts:setup]
```
:::

### Configure native currency symbol

If you need to display a native token symbol, such as showing how much gas a transaction requires, you should set the currency symbol to `USD` for Tempo as fees are denominated in USD.

### Dealing with fee token selection

On Tempo, users can pay fees in any supported stablecoin. You should quote gas/fee prices in your UI based on a transaction's fee token.

:::info
As a wallet developer, you can set the fee token for your user at the account level.

If you don't, Tempo uses a cascading fee token selection algorithm to determine the fee token for a transaction â€“ learn more about [Fee Token Preferences](/protocol/fees/spec-fee#fee-token-preferences).
:::

### Display token and network assets

Tempo provides a public tokenlist service that hosts token and network assets. You can pull these assets from our public tokenlist service to display in your UI.

- **GitHub**: [tempoxyz/tempo-apps/apps/tokenlist](https://github.com/tempoxyz/tempo-apps/tree/main/apps/tokenlist)
- **Tokenlist JSON**: [tempoxyz.github.io/tempo-apps/42429/tokenlist.json](https://tempoxyz.github.io/tempo-apps/42429/tokenlist.json)

### Integrate Tempo Transactions

We strongly recommend using [Tempo Transactions](/guide/tempo-transaction) if you control transaction submission. We have [SDKs and guides](/guide/tempo-transaction#integration-guides) available that should get you integrated in less than an hour!

:::tip
With Tempo Transactions, you can:
- Set the fee token for a transaction
- Sponsor user fees to enable feeless transaction experiences
- Batch multiple transactions together for higher throughput
- Execute transactions in parallel using independent nonces
- Schedule transactions to execute within a specific time window
:::
::::

## Recipes

### Get user's fee token

Retrieve the user's configured fee token preference:

```ts
import { getUserToken } from 'tempo.ts/viem'

const feeToken = await client.fee.getUserToken({ 
  account: userAddress 
})
```

See [`getUserToken`](/sdk/typescript/viem/fee.getUserToken) for full documentation.

### Get token balance

Check a user's balance for a specific token:

```ts
import { getBalance } from 'tempo.ts/viem'

const balance = await client.token.getBalance({ 
  account: userAddress,
  token: tokenAddress 
})
```

See [`getBalance`](/sdk/typescript/viem/token.getBalance) for full documentation.

### Set user fee token

Set the user's default fee token preference. This will be used for all transactions unless a different fee token is specified at the transaction level. 

```ts
import { setUserToken } from 'tempo.ts/viem'

await client.fee.setUserTokenSync({ 
  token: '0x20c0000000000000000000000000000000000001', 
})
```

See [`setUserToken`](/sdk/typescript/viem/fee.setUserToken) for full documentation.
