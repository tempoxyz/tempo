# `reward.cancel`

Cancels an active reward stream and refunds remaining tokens to the funder.

## Usage

Use the `reward.cancel` action on the Viem `client` to cancel a reward stream.

:::code-group

```ts twoslash [example.ts]
import { client, token } from './viem.config'

const { funder, id, receipt, refund } = await client.reward.cancelSync({
  id: 1n,
  token,
})

console.log('Refunded:', refund)
// @log: Refunded: 900000000000000000000n
console.log('Stream ID:', id)
// @log: Stream ID: 1n
```

```ts twoslash [viem.config.ts] filename="viem.config.ts"
// [!include ~/snippets/viem.config.ts:setup]
```

:::

### Asynchronous Usage

The example above uses a `*Sync` variant of the action, that will wait for the transaction to be included before returning.

If you are optimizing for performance, you should use the non-sync `reward.cancel` action and wait for inclusion manually:

```ts twoslash
import { Actions } from 'tempo.ts/viem'
import { client, token } from './viem.config'

const hash = await client.reward.cancel({
  id: 1n,
  token,
})
const receipt = await client.waitForTransactionReceipt({ hash })

const { args: { funder, id, refund } } 
  = Actions.reward.cancel.extractEvent(receipt.logs)
```

## Return Type

```ts
type ReturnType = {
  /** Address that funded the stream */
  funder: Address
  /** Stream ID that was canceled */
  id: bigint
  /** Transaction receipt */
  receipt: TransactionReceipt
  /** Amount refunded to the funder (0 if TIP-403 policy forbids transfer) */
  refund: bigint
}
```

## Parameters

### id

- **Type:** `bigint`

The unique stream ID to cancel. Must be owned by the caller.

### token

- **Type:** `Address`

Address of the TIP-20 token containing the reward stream.
