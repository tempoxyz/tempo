# TIP-1090: AWS Nitro Enclave Attestation Verification Precompile

## Abstract

This TIP introduces a precompile that verifies AWS Nitro Enclave attestation documents: it decodes the COSE_Sign1-signed CBOR document, validates the embedded X.509 certificate chain against the pinned AWS Nitro root, verifies the ECDSA P-384 signatures, and returns the signed fields (PCRs, module ID, timestamp, public key, user data, nonce). Callers enforce their own enclave identity and freshness policy.

## Motivation

[Tempo Zones](https://github.com/tempoxyz/zones) settle to Tempo through a portal that calls an enshrined verifier before releasing withdrawals from escrow. The [`IVerifier` interface](https://github.com/tempoxyz/zones/blob/main/specs/ref-impls/src/interfaces/IZone.sol) is proof-agnostic — the same state transition function runs in a ZKVM or a TEE — and the deployed verifier is currently a stub that accepts every batch. For the TEE backend, the prover runs in an AWS Nitro Enclave: the enclave signs the batch commitments with an enclave-held key, and a Nitro attestation document binds that key to the enclave image measurements (PCRs). Trusting that key onchain requires validating the attestation document — CBOR/COSE decoding, X.509 chain validation, SHA-384 hashing, and several ECDSA P-384 verifications — which existing EVM primitives cannot do at practical cost.

The precompile is zone-agnostic on purpose, so that the same primitive can serve any other TEE-backed integration without further protocol changes.

### Alternatives Considered

- **Pure Solidity** ([base/nitro-validator](https://github.com/base/nitro-validator)): P-384 via `MODEXP` costs ~267M gas per cold verification; with off-chain "inverse hint" witnesses it still needs ~27 KB of calldata per signature, five transactions, and a certificate cache contract. Prohibitive for continuous settlement.
- **ZK proof of attestation**: adds proving latency and off-chain tooling for what is a few signature verifications, and couples the TEE path to a proving stack it does not otherwise need.
- **Trusted oracle/committee**: reintroduces the trust the attestation is meant to remove.

A native precompile makes verification a single call, consistent with [TIP-1020](./tip-1020.md).

## Assumptions

- **AWS Nitro Attestation PKI is the root of trust.** The commercial-partition root, which is expiring in the year 2049 is embedded in the T9 precompile implementation; rotation will require a protocol upgrade.
- **No revocation checking.** AWS validator guidance disables CRLs, and Nitro PKI certificates are short-lived, bounding exposure. Applications needing revocation can key a registry on the returned leaf certificate hash.
- **Stable document format.** The precompile implements the [AWS attestation process specification](https://github.com/aws/aws-nitro-enclaves-nsm-api/blob/main/docs/attestation_process.md); format changes by AWS require a TIP update and hardfork.

## Threat Model

- **AWS (hypervisor and Attestation PKI)** — trusted. Root or intermediate compromise allows forgery; out of scope.
- **Enclave operator** — untrusted. Cannot forge PCRs, but can run *any* image and obtain valid attestations for it. The precompile proves *authenticity*, never *trustworthiness*; PCR/module ID policy is the calling contract's responsibility.
- **Transaction submitter** — untrusted. Documents in calldata are public and replayable; contracts MUST implement replay protection (nonce challenge, freshness window, one-time registration).
- **Malicious caller (DoS)** — untrusted. Mitigated by a hard input size cap, bounded collection sizes and nesting depth, and gas charged before verification work.
- **ECDSA malleability** — P-384 signatures are not guaranteed low-s and both forms are accepted (matching AWS). Contracts MUST NOT use raw attestation bytes or signatures as uniqueness keys.

# Specification

## Interface

The precompile exposes a single function (full Solidity definition in [Appendix A](#appendix-a-solidity-interface)):

```solidity
function verifyAttestation(bytes calldata document)
    external view returns (NitroAttestation memory attestation);
```

`document` is the raw signed attestation document as returned by the Nitro Security Module. On success it returns the parsed, validated contents, including the timestamp, PCRs, the optional `public_key`/`user_data`/`nonce` fields. On failure it reverts with `InvalidFormat`, `InvalidCertificate`, or `InvalidSignature` per the rules below.

The precompile enforces no application policy: callers MUST check the document values for validity themselves. Absent optional fields are returned as empty bytes.

## Validation Rules

Validation follows the [AWS attestation process specification](https://github.com/aws/aws-nitro-enclaves-nsm-api/blob/main/docs/attestation_process.md), sections 3.1–3.2. All checks MUST pass; any failure MUST revert with the indicated error.

### 1. Size limit

`document` MUST be at most **24,576 bytes** (the COSE payload is capped at 16,384 bytes by AWS; the limit allows for framing and signature). Violation: `InvalidFormat`.

### 2. COSE_Sign1 decoding

- The input MUST decode as a CBOR `COSE_Sign1` array of exactly 4 elements: `[protected, unprotected, payload, signature]`. A leading CBOR tag 18 is OPTIONAL; any other tag MUST be rejected.
- The protected header MUST decode to the map `{1: -35}` (ECDSA w/ SHA-384). The unprotected header MUST be ignored.
- The payload MUST be a byte string of 1–16,384 bytes.
- The signature MUST be exactly **96 bytes**, interpreted as `r || s` with 48-byte big-endian components (ES384; matches the NSM implementation and prior art).
- Trailing bytes after the COSE structure MUST be rejected.

Violation: `InvalidFormat`.

### 3. Payload (attestation document) validation

The payload MUST decode as a CBOR map. Mandatory fields: `module_id`, `digest`, `timestamp`, `pcrs`, `certificate`, `cabundle`. Optional: `public_key`, `user_data`, `nonce`. Recognized fields MUST NOT be null or appear more than once. Unrecognized keys MUST be skipped (they are covered by the AWS signature; this follows base/nitro-validator and preserves forward compatibility).

| Field | Type | Constraint |
|---|---|---|
| `module_id` | text string | non-empty |
| `digest` | text string | exactly `"SHA384"` |
| `timestamp` | unsigned integer | `> 0`, `≤ 2^64 − 1` |
| `pcrs` | map | 1–32 entries; keys: integers in `[0, 32)`, no duplicates; values: byte strings of length 32, 48, or 64 |
| `certificate` | byte string | 1–1,024 bytes, DER X.509 |
| `cabundle` | array | 1–32 entries, each a byte string of 1–1,024 bytes, DER X.509 |
| `public_key` | byte string (optional) | 1–1,024 bytes |
| `user_data` | byte string (optional) | 0–512 bytes |
| `nonce` | byte string (optional) | 0–512 bytes |

Definite- and indefinite-length CBOR MUST both be accepted, with size caps enforced before allocation. CBOR nesting deeper than **16 levels** MUST be rejected. Violation: `InvalidFormat`.

### 4. Certificate chain validation

`cabundle` is ordered `[ROOT, INTERM_1, …, INTERM_N]`; the verification path is:

```
certificate (leaf) ← INTERM_N ← … ← INTERM_1 ← ROOT
```

- `cabundle[0]` MUST be byte-identical (DER) to the pinned root. Otherwise: `InvalidCertificate`.
- Every certificate MUST be strict DER, X.509 v3, signature algorithm `ecdsa-with-SHA384` (OID `1.2.840.10045.4.3.3`), subject public key an uncompressed point on P-384 (`secp384r1`).
- Every validity window (`notBefore`, `notAfter`) MUST contain the current block timestamp.
- Issuer/subject linkage MUST hold at every step (compared on DER bytes).
- Basic constraints and key usage: root and intermediates `CA=true` with `keyCertSign` and any `pathLenConstraint` respected; leaf has `digitalSignature` and MUST NOT be a CA.
- Unrecognized *critical* extensions MUST be rejected. CRL/OCSP checking MUST NOT be performed (per AWS validator guidance).
- Each non-root certificate's signature MUST verify (ECDSA P-384 over SHA-384 of its `tbsCertificate`) under its parent's key. Signature failure: `InvalidSignature`; all other violations: `InvalidCertificate`.

### 5. Document signature verification

The COSE `Sig_structure` is constructed per RFC 9052:

```
[ "Signature1", protected, external_aad = h'', payload ]
```

The 96-byte signature MUST verify as ECDSA P-384 over the SHA-384 hash of the serialized `Sig_structure`, under the leaf certificate's public key. Both low-s and high-s signatures MUST be accepted. Failure: `InvalidSignature`.

### 6. Result

On success the precompile returns the ABI-encoded `NitroAttestation` struct. On any failure it reverts with the ABI-encoded error selector and MUST NOT return partial results.

## Gas Costs

All calls pay the standard precompile calldata cost of 6 gas per 32-byte word (rounded up), consistent with TIP-1020. Verification gas is charged in two stages, each **before** the corresponding work:

1. **Base cost** — `40,000` gas, charged before decoding. Covers CBOR/COSE parsing, payload validation, X.509 parsing, and SHA-384 hashing.
2. **Per-signature cost** — `35,000` gas per P-384 verification, charged after parsing determines the chain length. The verification count is `len(cabundle) + 1`.

A typical chain (root + 3 intermediates + leaf) performs 5 verifications: `40,000 + 5 × 35,000 = 215,000` gas plus calldata cost. Insufficient gas at either stage MUST fail with out-of-gas without performing the remaining work.

These numbers are provisional: before scheduling, the per-signature cost MUST be benchmarked against the node's P-384 implementation and calibrated against the existing P256 pricing (8,000 gas, TIP-1020).

## Compatibility

Additive: a new precompile at a previously empty address, activated at the `protocolVersion` hardfork. No existing opcodes, precompiles, or transaction formats change. Before activation, calls to the address behave as calls to an empty account.

Because certificate validity is checked against the block timestamp, the same document can verify in one block and fail in a later one (certificate expiry). Contracts needing a durable record should verify once and persist the result (e.g., register the enclave's `publicKey` after checking PCRs).

# Tooling

- **Reference implementation**: a `no_std` core crate implementing parsing and validation — `minicbor` for CBOR/COSE_Sign1 (pull-parser with explicit depth/size control), `x509-cert`/`der` for strict-DER X.509 (chain rules per §4 implemented directly, since the AWS profile fits no off-the-shelf validator) — generic over P-384 and SHA-384 backends. The node plugs in `aws-lc-rs` (`ECDSA_P384_SHA384`; Tempo already uses AWS-LC for P256); `no_std` consumers such as a ZKVM guest can plug in RustCrypto `p384`/`sha2`. The parser MUST be treated as consensus-critical and fuzzed, with `rustls-webpki`/`x509-parser` as differential-testing oracles in dev-dependencies.
- **Test fixtures**: vendored attestation documents (valid, and mutated per validation rule) plus a fixture root CA and generator, so tests do not depend on live AWS infrastructure or wall-clock time.
- **Reference consumer contract**: a minimal example of correct policy enforcement (PCR allowlist, freshness window, nonce challenge, one-time key registration). The zones `IVerifier` implementation is the first consumer.

# Observability

Node implementations SHOULD export:

- `tempo_nitro_attestation_verify_total{result="ok"|"invalid_format"|"invalid_certificate"|"invalid_signature"}` — surfaces failing integrations and anomalous spikes (e.g., certificate-expiry misconfiguration, parser probing).
- `tempo_nitro_attestation_verify_duration_seconds` — validates the gas schedule against real compute cost.

# Invariants

| ID | Invariant | Description |
|----|-----------|-------------|
| **NA1** | Determinism | For a fixed `(document, block timestamp, configured root)`, every node MUST produce an identical accept/reject result and identical output bytes. |
| **NA2** | Chain of trust | Verification MUST succeed only if the leaf chains to the pinned root with every X.509 check passing. |
| **NA3** | Signature soundness | Verification MUST succeed only if the COSE signature verifies under the leaf's P-384 key over the exact `Sig_structure` bytes. |
| **NA4** | Field completeness | Every mandatory-field, type, and size rule (§3) MUST be enforced; returned fields MUST be byte-identical to the signed payload contents. |
| **NA5** | Policy neutrality | Acceptance MUST be independent of PCR values, module ID, `timestamp` recency, and `public_key`/`user_data`/`nonce` contents (beyond structural constraints). Policy belongs to callers. |
| **NA6** | Revert on failure | Any validation failure MUST revert with the specified error; no partial results. |
| **NA7** | Gas before work | Base gas charged before parsing; per-signature gas before any P-384 verification. Insufficient gas MUST fail without doing the work. |
| **NA8** | Bounded resources | Input ≤ 24,576 bytes; ≤ 32 PCRs; ≤ 32 cabundle entries; nesting ≤ 16. Pathological inputs MUST be rejected before allocation-heavy work. |

The test suite MUST cover, at minimum: a known-good production attestation at a frozen block timestamp; tagged and untagged COSE; definite- and indefinite-length CBOR; each field constraint violated individually; duplicate keys/PCR indices; unknown payload fields (accepted) vs. unknown critical X.509 extensions (rejected); wrong root; broken linkage; expired/not-yet-valid certificates at boundary timestamps; wrong-curve and compressed-point keys; corrupted signatures; high-s acceptance; maximum-size inputs; out-of-gas at both charging stages.

# Appendix A: Solidity Interface

```solidity
interface INitroAttestationVerifier {
    /// @notice Input is structurally invalid: not valid CBOR/COSE, exceeds
    ///         size limits, missing mandatory fields, or violates constraints.
    error InvalidFormat();

    /// @notice Certificate chain invalid: root mismatch, broken linkage,
    ///         expired/not-yet-valid, wrong algorithm or key type, or a
    ///         constraints/key-usage/critical-extension violation.
    error InvalidCertificate();

    /// @notice A certificate or COSE document signature failed P-384 verification.
    error InvalidSignature();

    struct Pcr {
        /// @dev PCR index, in [0, 32).
        uint8 index;
        /// @dev PCR value; length is 32, 48, or 64 bytes.
        bytes value;
    }

    struct NitroAttestation {
        /// @dev Issuing Nitro hypervisor module ID (non-empty).
        string moduleId;
        /// @dev Document creation time, milliseconds since UNIX epoch (> 0).
        uint64 timestamp;
        /// @dev All locked PCRs, sorted ascending by index, no duplicates.
        Pcr[] pcrs;
        /// @dev Optional DER-encoded enclave public key; empty if absent.
        bytes publicKey;
        /// @dev Optional user data; empty if absent.
        bytes userData;
        /// @dev Optional nonce; empty if absent.
        bytes nonce;
        /// @dev SHA-256 of the DER-encoded leaf (signing) certificate.
        ///      Enables application-layer revocation registries.
        bytes32 leafCertHash;
    }

    /// @notice Verifies a signed AWS Nitro Enclave attestation document
    ///         (COSE_Sign1 CBOR, with or without leading CBOR tag 18):
    ///         decoding, field and size checks, chain validation to the pinned
    ///         AWS Nitro root, and P-384 verification of every signature.
    ///
    ///         The precompile enforces NO application policy. Callers MUST
    ///         check PCR values, module ID, timestamp freshness, and
    ///         nonce/user_data binding themselves.
    function verifyAttestation(bytes calldata document)
        external
        view
        returns (NitroAttestation memory attestation);
}
```
