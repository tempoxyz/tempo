# TIP-1036: T2 Hardfork Bug Fixes

## Abstract

This is a meta TIP that collects all audit-driven bug fixes, security hardening, and minor correctness improvements gated behind the T2 hardfork activation. Each change was identified through the Cyclops audit engagement or internal review and is individually small, but together they form the complete set of state-function-breaking fixes shipping in T2.

## Motivation

During the Cyclops audit cycle (SIGP + in-house reviews), several issues were identified across precompiles, the transaction pool, fee handling, and the account keychain. Individually these fixes do not warrant standalone TIPs, but they must be documented together as they collectively define the T2 STF delta beyond feature TIPs.

---

# Changes

## 1. Require `tx.origin` for AccountKeychain admin ops

**PR**: [#3202](https://github.com/tempoxyz/tempo/pull/3202)
T2 scopes `authorizeKey`, `revokeKey`, and `updateSpendingLimit` to require `msg_sender == tx_origin` and `transaction_key == Address::ZERO`. This prevents contract-mediated confused-deputy flows from reusing transaction-global root-key status to administer keys on behalf of the caller.

## 2. Reject self-sponsored fee payer signatures

**PR**: [#3200](https://github.com/tempoxyz/tempo/pull/3200)
Rejects AA transactions where the `fee_payer_signature` resolves back to the sender, preventing self-sponsored signatures from bypassing fee-payer assumptions. Enforced in both txpool validation and EVM fee-payer resolution.

## 3. Run invalidation eviction on reorg segments

**PR**: [#3171](https://github.com/tempoxyz/tempo/pull/3171)
On chain reorgs, invalidation events (key revocations, spending limit changes, validator/user token changes, blacklist additions) in the new canonical segment were silently ignored because maintenance continued before reaching `TempoPoolUpdates::from_chain()`. The fix extracts updates from the reorg segment and runs eviction for both the main and paused pools.

## 4. Remove unnecessary SLOAD in packed field structs

**PR**: [#2976](https://github.com/tempoxyz/tempo/pull/2976)
Skips the redundant `SLOAD` in the precompile `store` macro for the first packed slot of a struct. Since struct fields occupy their own slots (never packed with unrelated fields), starting from `U256::ZERO` instead of reading the existing value saves 1 cold SLOAD per packed-struct write.

## 5. Add `AccessKeySpend` event

**PR**: [#2914](https://github.com/tempoxyz/tempo/pull/2914)
Emits `AccessKeySpend(account, publicKey, token, amount, remainingLimit)` from `verify_and_update_spending` whenever an access key's spending limit is deducted. Only fires for actual access-key spends — root key transactions return early before emission.

## 6. Check token paused in internal DEX balance swaps

**PR**: [#3204](https://github.com/tempoxyz/tempo/pull/3204)
Adds a `check_not_paused()` call in `StablecoinDEX` internal balance transfers gated behind `is_t2()`. Previously, swaps using internal DEX balances could bypass the token pause state.

## 7. Correct built-in policy type data for TIP403Registry

**PR**: [#3203](https://github.com/tempoxyz/tempo/pull/3203)
Built-in policies (`REJECT_ALL` / `ALLOW_ALL`) are virtual and not stored on-chain. On T2, `policyData()` now returns the correct `PolicyType` (`WHITELIST` / `BLACKLIST` respectively) and `Address::ZERO` admin for these built-in IDs instead of falling through to storage reads.

## 8. Guard `normalize_p256_s` against out-of-range values

**PR**: [#3101](https://github.com/tempoxyz/tempo/pull/3101)
Changes `normalize_p256_s` to return `Option<B256>`, rejecting `s == 0` and `s >= P256_ORDER` before subtraction. Without this guard, unsigned subtraction wraps in release mode and can produce an invalid scalar.

## 9. Resolve compound policy sub-policy IDs for eviction

**PR**: [#3196](https://github.com/tempoxyz/tempo/pull/3196) *(merged)*
Fixes txpool eviction to correctly resolve sub-policy IDs within compound policies, ensuring that blacklist additions propagate to transactions authorized through compound policy chains.

## 10. Strict payment calldata validation in pool and builder

**PR**: [#3099](https://github.com/tempoxyz/tempo/pull/3099) *(merged)*

Adds type-safe calldata matching for payment lane classification using `SolCall::SELECTOR` and `SolType::ENCODED_SIZE`, preventing transactions with unrecognized or oversized calldata from being classified as payments and accessing the payment lane gas budget.

# Invariants

All existing protocol invariants are preserved. Each fix either tightens validation (rejecting previously-accepted invalid states) or corrects event emission / storage reads without introducing new state transitions. The key invariants reinforced:

1. **Admin-op isolation**: AccountKeychain admin operations can only be invoked by the account owner via a direct transaction (not via contract call).
2. **Fee-payer distinctness**: The fee payer of an AA transaction must be a different account than the sender.
3. **Reorg consistency**: Pool eviction runs on the new canonical segment after every reorg.
4. **Token pause enforcement**: Paused tokens cannot be moved through any path, including internal DEX balances.
5. **Policy correctness**: Built-in policy queries return semantically correct types.
6. **Signature scalar validity**: P256 `s` values are range-checked before normalization.
